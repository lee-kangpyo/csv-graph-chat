## Context

- 현재 차트 관련 질문 시 프론트엔드가 `/api/chat/stream`(SSE)과 `/api/chart/generate`(JSON)를 동시에 호출하지만, LLM API의 동시 요청 제한(429 에러)으로 차트 생성이 실패한다.
- 기존 `insight_recommendation.py`의 `generate_insight_prompt()`는 이미 AI에게 2~3개의 분석 질문을 추천하도록 지시하고 있으나, 출력이 자유 텍스트이므로 프론트엔드에서 구조적으로 파싱하기 어렵다.
- `generate_chart_prompt()`는 단일 차트 Intent JSON을 반환하며, `pandas_processor.py`가 11종의 차트 타입을 처리할 수 있다.
- `chatStore.js`는 `{ role, content, timestamp }` 구조로 메시지를 저장하며, 추천 데이터를 별도로 관리하는 필드가 없다.

## Goals / Non-Goals

### Goals

- AI가 차트 관련 질문에 대해 **2~3개의 구조화된 차트 추천 항목**(차트 타입 + 제목 + 설명)을 SSE 텍스트 응답에 포함하여 전달한다.
- 프론트엔드가 SSE 스트리밍 중 추천 태그를 실시간 파싱하여 **클릭 가능한 Action Chip 버튼**으로 렌더링한다.
- Action Chip 클릭 시 해당 차트만 `/api/chart/generate`를 호출하여 바스켓에 저장한다.
- "다 그려줘" 명령 시 이전 추천 리스트를 참조하여 **순차적으로** 여러 차트를 생성한다.

### Non-Goals

- 대화 히스토리를 백엔드 세션/DB에 저장하는 서버 사이드 컨텍스트 관리 (컨텍스트는 프론트엔드에서만 관리)
- 추천 항목 수를 사용자가 설정하는 기능
- 차트 품질/추천 정확도의 근본적 개선 (별도 변경)

## Decisions

### 1) 추천 데이터 전달 방식: SSE 텍스트 내 구조화 태그

**결정**: AI 응답 텍스트 본문 끝에 `[SUGGESTION:{"title":"...", "chart_type":"...", "description":"..."}]` 형태의 태그를 포함한다.

**대안 검토**:
- A) SSE에 별도 `suggestion` 이벤트 타입 추가 → 백엔드에서 LLM 응답을 파싱하여 별도 이벤트로 분리해야 하므로 복잡도 증가
- B) 응답 완료 후 별도 API 호출 → 추가 LLM 호출이 필요하여 비용/시간 증가
- **C) 텍스트 내 태그 (선택)** → 프롬프트만 수정하면 되고, 프론트에서 정규식으로 파싱. 가장 단순하며 기존 SSE 인프라를 그대로 사용

**근거**: LLM이 이미 추천 텍스트를 생성하므로, 출력 형식만 구조화하면 추가 API 호출 없이 달성 가능하다.

### 2) 프론트엔드 맥락 관리: chatStore 확장

**결정**: `chatStore`의 메시지 구조에 `suggestions` 배열 필드를 추가하여, AI 메시지가 어떤 추천을 했는지 프론트엔드에서 기억한다.

```js
{ role: 'ai', content: '...', timestamp: ..., suggestions: [{title, chart_type, description, message_context}] }
```

**근거**: "다 그려줘" 처리 시 직전 AI 메시지의 `suggestions`를 순회하면 별도 서버 세션 없이 구현 가능하다.

### 3) 일괄 생성("다 그려줘") 처리: 프론트엔드 순차 호출

**결정**: 프론트엔드가 `suggestions` 배열을 순회하며 `/api/chart/generate`를 하나씩 `await`로 호출한다.

**대안 검토**:
- A) 백엔드에 배치 엔드포인트 추가(`POST /api/chart/generate-batch`) → 하나의 요청이 오래 걸리면 타임아웃 위험
- **B) 프론트엔드 순차 호출 (선택)** → 각 요청이 독립적이므로 실패 시 나머지는 계속 진행 가능. Rate Limit을 자연스럽게 회피

**근거**: 프론트에서 1개씩 호출하면 LLM API 제한을 완벽히 회피하면서도, 바스켓에 하나씩 추가되는 진행 상황을 사용자가 실시간으로 볼 수 있다.

### 4) 추천 태그 파싱 전략: 스트리밍 중 버퍼 기반 파싱

**결정**: SSE 청크 수신 중 `[SUGGESTION:` 패턴이 감지되면 버퍼링을 시작하고, `]`로 닫힐 때 JSON을 파싱하여 추천 항목으로 확정한다. 파싱된 태그는 화면에 텍스트로 표시하지 않고 `suggestions` 배열에만 추가한다.

### 5) "다 그려줘" 감지: 프론트엔드 키워드 매칭

**결정**: 기존 `CHART_KEYWORDS` 패턴을 확장하여 "다 그려", "전부 그려", "모두 그려", "다 만들어" 등의 일괄 생성 키워드를 별도 리스트로 관리한다. 일괄 키워드가 감지되면 직전 AI 메시지의 `suggestions`를 모두 순차 실행한다.

## Risks / Trade-offs

- **[LLM 출력 불안정]** → LLM이 `[SUGGESTION:...]` 태그를 정확히 생성하지 못할 수 있다. → **완화**: 태그 파싱 실패 시 무시하되 텍스트 응답 자체는 정상 표시. fallback으로 기존 자동 생성 로직 유지 가능.
- **[SSE 청크 분할]** → `[SUGGESTION:...]` 태그가 여러 SSE 청크에 걸쳐 분할될 수 있다. → **완화**: 버퍼 기반 파싱으로 `[SUGGESTION:` 시작부터 `]` 종료까지 버퍼링.
- **[일괄 생성 시간]** → 3개 차트 순차 생성 시 체감 대기시간 증가. → **완화**: 각 차트 완료 시마다 바스켓에 즉시 반영하여 진행 상황을 시각적으로 표시.

## Open Questions

- 추천 태그 파싱이 실패했을 때, 기존의 자동 차트 생성으로 fallback할지 아니면 텍스트 응답만 표시할지 (1차에서는 텍스트만 표시 권장)
