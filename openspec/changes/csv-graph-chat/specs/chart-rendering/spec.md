## ADDED Requirements

### Requirement: Chart.js rendering
The system SHALL render charts using Chart.js library loaded from CDN.

#### Scenario: Render line chart
- **WHEN** AI generates line chart configuration
- **THEN** system SHALL render using Chart.js with type "line"
- **AND** display in the graph area

#### Scenario: Render bar chart
- **WHEN** AI generates bar chart configuration
- **THEN** system SHALL render using Chart.js with type "bar"

#### Scenario: Render doughnut chart
- **WHEN** AI generates doughnut chart configuration
- **THEN** system SHALL render using Chart.js with type "doughnut"

#### Scenario: Render scatter chart
- **WHEN** AI generates scatter chart configuration
- **THEN** system SHALL render using Chart.js with type "scatter"

### Requirement: Real-time chart update
The system SHALL update the chart in real-time when AI sends modified configuration.

#### Scenario: Chart updates during conversation
- **WHEN** user requests a modification
- **THEN** system SHALL update the existing chart without page reload
- **AND** maintain smooth transition

### Requirement: Chart responsive sizing
The system SHALL make charts responsive to container size.

#### Scenario: Responsive chart
- **WHEN** chart container resizes (window resize, sidebar toggle)
- **THEN** chart SHALL automatically adjust to fit the container
- **AND** maintain aspect ratio

### Requirement: Chart.js configuration from AI
The system SHALL accept Chart.js configuration objects generated by AI.

#### Scenario: Accept full Chart.js config
- **WHEN** AI sends { type, data, options }
- **THEN** system SHALL pass this directly to new Chart(ctx, config)

#### Scenario: Missing options
- **WHEN** AI sends config without options
- **THEN** system SHALL apply default options for responsiveness and tooltips

### Requirement: Chart download as HTML
The system SHALL export charts as standalone HTML files.

#### Scenario: HTML export includes data
- **WHEN** user downloads chart as HTML
- **THEN** HTML SHALL include:
- **AND** Chart.js from CDN
- **AND** embedded chart configuration
- **AND** data values embedded (not external reference)

#### Scenario: HTML is viewable offline
- **WHEN** user opens downloaded HTML file without internet
- **THEN** chart SHALL render correctly (if CDN is cached)
