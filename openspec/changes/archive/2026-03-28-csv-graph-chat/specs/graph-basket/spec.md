## ADDED Requirements

### Requirement: 바구니에 그래프 저장
시스템은 사용자가 현재 그래프 설정을 나중에 다운로드하기 위해 바구니에 저장할 수 있어야 한다.

#### Scenario: 기본 이름으로 그래프 저장
- **WHEN** 사용자가 "바구니에 담기" 버튼을 클릭하면
- **THEN** 시스템은 콘텐츠 기반의 자동 생성된 이름으로 현재 그래프 설정을 저장해야 한다
- **AND** 토스트 성공 메시지 "바구니에 담겼어요!"를 표시해야 한다
- **AND** 바구니 카운트를 업데이트해야 한다

#### Scenario: 저장 실패
- **WHEN** 사용자가 "바구니에 담기"를 클릭했지만 저장 작업이 실패하면
- **THEN** 시스템은 토스트 에러 "저장에 실패했어요. 다시 시도해주세요"를 표시해야 한다
- **AND** 바구니를 업데이트하지 않아야 한다

### Requirement: 바구니 내용 보기
시스템은 바구니에 저장된 모든 그래프를 보여주는 사이드바를 표시해야 한다.

#### Scenario: 바구니 목록 표시
- **WHEN** 사용자가 바구니에 저장된 그래프를 가지고 있으면
- **THEN** 시스템은 사이드바에 다음을 보여줘야 한다:
- **AND** 이름과 타임스탬프가 있는 각 그래프 카드
- **AND** 미리보기 버튼(눈 아이콘)
- **AND** 삭제 버튼(휴지통 아이콘)

#### Scenario: 빈 바구니
- **WHEN** 바구니가 비어있으면
- **THEN** 시스템은 "바구니가 비어있어요"를 표시해야 한다
- **AND** 모두 다운로드 버튼을 숨겨야 한다

### Requirement: 바구니에서 그래프 미리보기
시스템은 사용자가 저장된 그래프를 메인 그래프 영역에서 미리볼 수 있어야 한다.

#### Scenario: 그래프 미리보기
- **WHEN** 사용자가 바구니 항목의 미리보기 버튼을 클릭하면
- **THEN** 시스템은 해당 그래프 설정을 메인 차트 영역에 로드해야 한다
- **AND** ECharts로 표시해야 한다

### Requirement: 바구니에서 그래프 삭제
시스템은 사용자가 바구니에서 개별 그래프를 삭제할 수 있어야 한다.

#### Scenario: 개별 그래프 삭제
- **WHEN** 사용자가 바구니 항목의 삭제 버튼을 클릭하면
- **THEN** 시스템은 해당 그래프를 바구니에서 제거해야 한다
- **AND** 바구니 카운트를 업데이트해야 한다
- **AND** 사이드바 목록을 업데이트해야 한다

### Requirement: HTML로 모든 그래프 다운로드
시스템은 사용자가 모든 바구니 그래프를 단일 HTML 파일으로 다운로드할 수 있어야 한다.

#### Scenario: 모든 그래프 다운로드
- **WHEN** 사용자가 "모두 다운로드" 버튼을 클릭하면
- **THEN** 시스템은 다음을 포함한 HTML 파일을 생성해야 한다:
- **AND** ECharts로 렌더링된 모든 저장된 그래프
- **AND** 원본 CSV 데이터 임베디드 또는 링크
- **AND** 브라우저 다운로드 트리거

#### Scenario: 개별 그래프 HTML 다운로드
- **WHEN** 사용자가 개별 그래프를 다운로드하려면
- **THEN** 시스템은 해당 그래프만 있는 HTML을 생성해야 한다
- **AND** 브라우저 다운로드 트리거

### Requirement: 바구니 데이터 지속성
시스템은 바구니 데이터를 서버 측에 유지해야 한다.

#### Scenario: 데이터 지속성
- **WHEN** 사용자가 그래프를 바구니에 저장하면
- **THEN** 시스템은 데이터베이스에 다음을 저장해야 한다:
- **AND** 그래프 설정(JSON)
- **AND** 관련 CSV 참조 또는 데이터
- **AND** 생성 타임스탬프

#### Scenario: 세션 없는 작업
- **WHEN** 사용자가 바구니에 저장된 그래프를 가지고 있고
- **AND** 사용자가 페이지를 새로고침하면
- **THEN** 바구니는 데이터가 서버에 유지되므로 계속 채워져 있어야 한다
