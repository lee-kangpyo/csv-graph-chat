## Context

현재 React 앱은 `ChatArea`와 `ChatInput` 사이에 생성 그래프를 인라인으로 표시한다. ECharts로 렌더링하며 그래프당 세로 약 400~600px를 사용해 대화 영역이 줄어든다. 상태는 Zustand(`chatStore`, `csvStore`, `basketStore`)를 쓰고, 바구니 API(`POST /api/basket/`)는 이미 있다. `GraphView`는 `App.jsx`에 연결되어 있으나 바구니 저장과는 연동되지 않았다.

## Goals / Non-Goals

**Goals:**
- 인라인 `GraphView` 제거로 채팅 영역 세로 공간 확보(약 400~600px)
- 사용자 개입 없이 채팅에서 생성된 모든 그래프를 바구니에 자동 저장
- 바구니 항목 클릭 시 화면 약 80% 모달로 그래프·사용자 질문 표시
- 바구니 사이드바 항목에 미니 썸네일 미리보기
- ECharts 렌더링 품질 유지

**Non-Goals:**
- 백엔드 API 엔드포인트 변경(기존 `POST /api/basket/` 사용)
- 채팅 기록·CSV 업로드 동작 변경
- 모달 안에서 그래프 편집
- 바구니 HTML보내기 기능 변경

## Decisions

### 모달 크기·배치
**결정:** 화면 너비·높이 각 80%, 중앙, 딤 오버레이  
**이유:** 복잡한 차트도 식별 가능하고, 전체 화면보다 부담이 적다.

### 자동 바구니 저장
**결정:** `ChatInput`에서 그래프 생성 직후 바구니 API 호출  
**이유:** 그래프 유실 방지, 별도 저장 버튼 불필요. 낙관적 UI는 `basketStore`로 처리.

### 바구니 미리보기
**결정:** 각 `BasketItem`에 약 150×100px 소형 ECharts 인스턴스  
**이유:** 별도 이미지 API 없이 시각적 구분 가능, ECharts 다중 인스턴스는 일반적으로 감당 가능.

### 모달 내 그래프
**결정:** `GraphModal` 신설, `GraphView`와 동일한 ECharts 설정 처리 재사용  
**이유:** 중복 최소화, 렌더링 일관성.

### 상태
**결정:** `App.jsx`의 `currentGraph` 제거, 그래프는 바구니(API·스토어)를 단일 출처로 사용  
**이유:** 흐름 단순화, 중복 상태 제거.

## Risks / Trade-offs

- **성능**: 바구니에 그래프가 많으면 ECharts 인스턴스가 많아질 수 있음 → 항목 수 제한·가상 스크롤·50개 이상 수동 테스트 검토.
- **API 지연**: 자동 저장이 응답을 지연시킬 수 있음 → 낙관적 업데이트, 실패 시 롤백.
- **z-index**: 모달과 토스트 등 겹침 → 모달 우선(예: 1000), 토스트(예: 999) 등 계층 명시.
- **인라인 제거 혼란**: “그래프가 어디 갔나” → 바구니 안내 문구(온보딩)로 완화.
- **`currentGraph` 제거**: 다른 컴포넌트 의존 시 깨질 수 있음 → 참조 제거·테스트로 검증.

## Migration Plan

**배포:**
1. 백엔드 변경 없음
2. 프론트엔드에 신규 컴포넌트·라우팅 반영
3. 기존 사용자 바구니는 비어 있을 수 있음(이전에는 그래프가 일시적이었음)

**롤백:**
- 프론트 이전 버전으로 되돌리기
- 이미 바구니에 저장된 데이터는 유지
- 인라인 `GraphView` 복구

**데이터 마이그레이션:** 없음(과거 인라인 그래프는 영속 저장되지 않음)
