## MODIFIED Requirements

### Requirement: AI 인사이트 추천
CSV 데이터를 분석한 후, 시스템은 AI가 계산된 실제 통계(Data Profile)를 기반으로 사실 근거 있는 인사이트를 제공해야 한다.

#### Scenario: 데이터 타입 기반 AI 추천
- **WHEN** CSV에 Date + Number 컬럼이 있으면
- **THEN** AI는 "시계열 추이"를 추천해야 한다
- **AND** 월별/분기별/연간 비교 옵션을 제시해야 한다

#### Scenario: 그룹 비교 추천
- **WHEN** CSV에 Category + Number 컬럼이 있으면
- **THEN** AI는 "그룹별 비교"를 추천해야 한다
- **AND** 막대 그래프나 파이 차트를 제안해야 한다

#### Scenario: 직접 요청 옵션 제공
- **WHEN** AI가 추천 인사이트를 제시하면
- **THEN** "직접 설명해주세요" 옵션도 함께 제시해야 한다
- **AND** 사용자의 직접적인 요청을 기다려야 한다

#### Scenario: Data Profile 기반 이상치 인사이트 제공
- **WHEN** 사용자가 데이터 이상 여부를 질문하거나 AI가 인사이트를 제공할 때
- **THEN** AI는 실제 계산된 이상치 개수와 해당 컬럼명을 응답에 포함해야 한다
- **AND** 추측이 아닌 Data Profile의 수치를 인용해야 한다

#### Scenario: 상관관계 인사이트에 실제 계수 포함
- **WHEN** 사용자가 컬럼 간 연관성을 질문하거나 AI가 자발적으로 상관관계를 언급할 때
- **THEN** AI는 Pearson 상관계수 값을 함께 제시해야 한다
- **AND** 계수 절댓값이 0.5 미만인 관계는 "약한 관계"로 명시해야 한다

#### Scenario: 프롬프트 토큰 예산 준수 (컬럼 10개 이하)
- **WHEN** CSV 컬럼 수가 10개 이하이고 Data Profile이 존재하면
- **THEN** 시스템은 모든 컬럼의 통계를 System 프롬프트에 포함해야 한다

#### Scenario: 프롬프트 토큰 예산 준수 (컬럼 10개 초과)
- **WHEN** CSV 컬럼 수가 10개를 초과하고 Data Profile이 존재하면
- **THEN** 시스템은 이상치가 가장 많은 컬럼 5개와 상관관계 상위 5쌍과 사용자 질문과 연관된 컬럼의 통계만 System 프롬프트에 포함해야 한다

#### Scenario: 분석 프레임워크 순서 준수
- **WHEN** AI가 인사이트 응답을 생성할 때
- **THEN** AI는 이상치 확인 → 추세 분석 → 상관관계 → 세그먼트 → 액션 권고 순서로 분석을 전개해야 한다

## ADDED Requirements

### Requirement: 질문 연관 통계 선별 주입
시스템은 사용자 질문에서 언급된 컬럼명을 감지하고, 해당 컬럼의 통계를 프롬프트 내 우선 배치해야 한다.

#### Scenario: 질문에 컬럼명이 포함된 경우
- **WHEN** 사용자 질문에 CSV 컬럼명과 일치하는 단어가 포함되어 있으면
- **THEN** 시스템은 해당 컬럼의 통계를 Data Profile로부터 추출하여 프롬프트 상단에 배치해야 한다

#### Scenario: 질문에 컬럼명이 없는 경우
- **WHEN** 사용자 질문에서 컬럼명이 감지되지 않으면
- **THEN** 시스템은 Data Profile의 이상치 요약과 최상위 상관관계 쌍을 기본으로 주입해야 한다
