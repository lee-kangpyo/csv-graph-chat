## Purpose

사용자가 "다 그려줘" 등 일괄 생성을 요청할 때, 이전 추천 리스트를 기반으로 여러 차트를 순차적으로 생성하는 기능.

## Requirements

### Requirement: 일괄 차트 생성 명령 감지
프론트엔드는 사용자의 "다 그려줘" 류의 일괄 생성 명령을 감지하여야 한다.

#### Scenario: 일괄 생성 키워드 감지
- **WHEN** 사용자 메시지에 "다 그려", "전부 그려", "모두 그려", "다 만들어", "전부 만들어" 중 하나가 포함될 때
- **AND** 직전 AI 메시지에 1개 이상의 추천 항목(`suggestions`)이 존재할 때
- **THEN** 프론트엔드는 일괄 차트 생성 모드로 진입하여야 한다

#### Scenario: 추천 없이 일괄 생성 시도
- **WHEN** 사용자가 일괄 생성 명령을 보내지만 직전 AI 메시지에 추천 항목이 없을 때
- **THEN** 일반 대화로 처리하여야 한다(일괄 생성 모드 진입하지 않음)

### Requirement: 순차적 일괄 차트 생성
일괄 생성 모드에서 프론트엔드는 추천된 차트들을 순차적으로 생성하여야 한다.

#### Scenario: 추천 리스트 순차 생성
- **WHEN** 일괄 차트 생성 모드에 진입할 때
- **THEN** 직전 AI 메시지의 `suggestions` 배열을 순서대로 순회하여야 한다
- **AND** 각 추천 항목에 대해 `/api/chart/generate`를 하나씩 호출하여야 한다(동시 호출 금지)
- **AND** 각 차트 생성 시작 전에 바구켓에 로딩 placeholder를 추가하여야 한다
- **AND** 각 차트 생성 완료 시 placeholder를 실제 차트로 교체하고 `POST /api/basket/`로 저장하여야 한다

#### Scenario: 일괄 생성 중 개별 실패 처리
- **WHEN** 일괄 생성 중 특정 차트 생성이 실패할 때
- **THEN** 해당 항목의 placeholder를 제거하고 오류를 표시하여야 한다
- **AND** 나머지 차트 생성은 계속 진행하여야 한다(전체 중단하지 않음)

#### Scenario: 일괄 생성 진행 상태 표시
- **WHEN** 일괄 차트 생성이 진행 중일 때
- **THEN** 사용자에게 진행 상황(예: "2/3 차트 생성 중...")을 표시하여야 한다
- **AND** 모든 차트 생성이 완료되면 완료 메시지를 표시하여야 한다
