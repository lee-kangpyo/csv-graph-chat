## Purpose
생성된 그래프를 사용자의 수동 저장 없이 즉시 바구니에 자동 저장하고, 실패 시 적절히 처리하는 시스템

## Requirements

### Requirement: 생성 그래프 자동 바구니 저장
시스템은 채팅에서 생성된 그래프를 사용자의 수동 저장 없이 즉시 바구니에 저장해야 한다.

#### Scenario: 채팅에서 그래프 생성 시 저장
- **WHEN** AI가 사용자 질문에 따라 그래프를 생성할 때
- **THEN** 시스템은 즉시 `POST /api/basket/`로 그래프 설정을 전송한다
- **AND** 그래프 이름은 사용자 질문에서 유도한다(최대 50자)
- **AND** `graph_config`에는 완전한 ECharts 설정이 포함된다
- **AND** 사용자 질문이 그래프와 연결되어 저장된다
- **AND** 별도 확인 단계는 필요 없다

### Requirement: 바구니 저장 실패 처리
시스템은 자동 저장이 실패해도 앱이 깨지지 않도록 처리해야 한다.

#### Scenario: 바구니 API 실패
- **WHEN** 자동 저장이 실패할 때(네트워크·서버 오류)
- **THEN** UI에 오류 메시지가 표시된다
- **AND** 사용자가 모달 등으로 그래프를 볼 수 있다면 그 표시는 가능한 범위에서 유지된다
- **AND** 오류로 인해 새 그래프 생성이 막히지 않는다
- **AND** 사용자는 재시도하거나 다음 질문을 계속할 수 있다

#### Scenario: 바구니 저장 성공
- **WHEN** 바구니 API가 성공 응답을 반환할 때
- **THEN** 바구니 사이드바에 새 그래프가 반영된다
- **AND** 새 항목에 미니 미리보기가 렌더링된다
- **AND** 모달 보기용으로 그래프에 접근할 수 있다

### Requirement: 낙관적 바구니 갱신
시스템은 저장 요청이 진행 중일 때도 바구니 UI를 가능한 한 즉시 갱신해야 한다(SHOULD).

#### Scenario: 생성 직후 바구니에 즉시 반영
- **WHEN** 채팅에서 그래프가 생성될 때
- **THEN** 바구니 사이드바에 새 그래프가 즉시 나타난다(낙관적 업데이트)
- **AND** API가 성공하면 해당 상태가 유지된다
- **AND** API가 실패하면 UI에서 해당 항목이 제거되고 오류가 표시된다

### Requirement: 그래프 이름 생성
시스템은 자동 저장되는 그래프에 대해 사용자 질문을 기반으로 설명적인 이름을 만들어야 한다.

#### Scenario: 질문에서 이름 유도
- **WHEN** 사용자가 "Show me sales by month"처럼 질문할 때
- **THEN** 그래프 이름은 해당 질문이거나 50자로 잘린 형태가 된다
- **AND** 원 질문의 의미가 이름에 남는다
- **AND** 특수문자는 안전하게 처리된다
- **AND** 이름은 BasketItem에 표시된다

#### Scenario: 일반적인 질문용 폴백 이름
- **WHEN** 사용자가 "Create a graph"처럼 구체적이지 않은 질문을 할 때
- **THEN** 시스템은 `Graph [timestamp]` 또는 `Graph [N]` 형태의 폴백 이름을 쓴다
- **AND** 바구니 내에서 이름이 구분 가능하다
