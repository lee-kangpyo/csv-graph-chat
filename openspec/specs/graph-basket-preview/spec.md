## Purpose
바구니 사이드바 항목마다 그래프의 축소 미리보기를 표시하는 시스템

## Requirements

### Requirement: BasketItem 썸네일 미리보기
시스템은 바구니 사이드바 항목마다 그래프의 축소 미리보기를 표시하여 사용자가 유형을 시각적으로 구분할 수 있어야 한다.

#### Scenario: BasketItem에 미니 그래프 미리보기
- **WHEN** 바구니에 저장된 그래프가 있을 때
- **THEN** 각 BasketItem에 미니 그래프 미리보기(150×100px)가 표시된다
- **AND** 미리보기는 꺾은선·막대 등 그래프 유형을 알아볼 수 있게 한다
- **AND** 미리보기는 축소된 설정으로 ECharts를 사용한다
- **AND** 미리보기는 그래프 이름 옆 또는 위에 배치된다

### Requirement: 미리보기 렌더링 성능
시스템은 BasketItem 미리보기가 앱 전체 성능에 과도한 부담을 주지 않도록 효율적으로 렌더링해야 한다.

#### Scenario: 다수 미리보기가 부드럽게 동작
- **WHEN** 바구니에 그래프가 20개 이상 있을 때
- **THEN** 모든 미리보기가 눈에 띄는 지연 없이 렌더링된다
- **AND** 바구니 스크롤이 가능한 한 부드럽다(목표 60fps)
- **AND** 메모리 사용이 허용 가능한 범위에 있다

### Requirement: 미리보기 클릭 동작
시스템은 미리보기가 있어도 BasketItem 클릭(미리보기 포함)으로 전체 그래프 보기가 열려야 한다.

#### Scenario: 미리보기 클릭 시 전체 모달
- **WHEN** 바구니 항목이 **로딩 placeholder** 상태(`graph_config`가 아직 없음)일 때
- **THEN** 사용자가 항목(미리보기 포함)을 클릭해도 전체 그래프 모달은 열려서는 안 된다
- **AND** 로딩 상태임을 사용자가 인지할 수 있는 UI(스피너/문구)가 유지되어야 한다

- **WHEN** 바구니 항목이 최종 저장되어 `graph_config`가 존재할 때
- **THEN** 사용자가 그래프 미리보기 썸네일을 클릭할 때
- **THEN** 화면 약 80% 크기의 전체 그래프 모달이 열린다
- **AND** 모달에는 완전한 그래프와 사용자 질문이 표시된다
- **AND** 동작은 항목 텍스트 영역 클릭과 동일하다
