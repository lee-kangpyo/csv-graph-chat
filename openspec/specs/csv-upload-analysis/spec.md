## Purpose
CSV 파일 업로드, 자동 헤더 감지, 데이터 타입 탐지, AI 컬럼명 추론 및 사용자 수정 기능을 제공하는 데이터 입력 시스템

## Requirements

### Requirement: CSV 파일 업로드
시스템은 사용자가 CSV 파일을 프론트엔드 인터페이스를 통해 업로드할 수 있어야 한다. 파일 크기는 10MB를 초과하지 않아야 한다.

#### Scenario: 파일 업로드 성공
- **WHEN** 사용자가 CSV 파일을 선택하고 업로드를 클릭하면
- **THEN** 시스템은 파일이 10MB 미만이고 유효한 CSV인지 검증해야 한다
- **AND** CSV 데이터와 감지된 컬럼을 반환해야 한다

#### Scenario: 파일 크기 초과
- **WHEN** 사용자가 10MB보다 큰 CSV 파일을 선택하면
- **THEN** 시스템은 토스트 에러 메시지 "파일이 너무 커요 (10MB 이하로 올려주세요)"를 표시해야 한다
- **AND** 업로드를 거부해야 한다

#### Scenario: 잘못된 파일 형식
- **WHEN** 사용자가 CSV가 아닌 파일을 선택하면
- **THEN** 시스템은 토스트 에러 메시지 "CSV 파일만 업로드 가능합니다"를 표시해야 한다
- **AND** 업로드를 거부해야 한다

### Requirement: 자동 헤더 감지
시스템은 CSV에 의미 있는 헤더가 있는지 또는 헤더가 없는지(첫 번째 행이 데이터)를 자동으로 감지해야 한다.

#### Scenario: 의미 있는 헤더가 있는 CSV
- **WHEN** CSV가 "date", "sales", "region" 같은 헤더를 포함하면
- **THEN** 시스템은 해당 헤더를 컬럼명으로 사용해야 한다

#### Scenario: 의미 없는 헤더가 있는 CSV
- **WHEN** CSV가 "a", "b", "c" 같은 헤더를 포함하면
- **THEN** 시스템은 헤더가 없는 것처럼 처리해야 한다

#### Scenario: 헤더가 없는 CSV
- **WHEN** CSV에 헤더가 없고(첫 번째 행이 데이터)
- **THEN** 시스템은 첫 번째 행을 데이터로 처리해야 한다
- **AND** "col_1", "col_2" 같은 임시 이름을 할당해야 한다

### Requirement: AI 컬럼명 추론
시스템은 AI를 사용하여 컬럼 데이터를 분석하고 데이터 패턴에 기반한 의미 있는 컬럼명을 제안해야 한다.

#### Scenario: AI가 컬럼 의미 추론 성공
- **WHEN** AI가 "2024-01", "2024-02" 같은 날짜 패턴이 있는 컬럼을 분석하면
- **THEN** AI는 컬럼명으로 "date" 또는 "날짜"를 제안해야 한다

#### Scenario: AI가 숫자 컬럼 감지
- **WHEN** AI가 "100", "150", "200" 같은 숫자 값이 있는 컬럼을 분석하면
- **THEN** AI는 컬럼명으로 "sales" 또는 "매출"을 제안해야 한다

#### Scenario: AI가 범주형 컬럼 감지
- **WHEN** AI가 "Seoul", "Busan", "Daegu" 같은 반복되는 문자열 값이 있는 컬럼을 분석하면
- **THEN** AI는 컬럼명으로 "region" 또는 "지역"을 제안해야 한다

### Requirement: 사용자 컬럼명 수정
시스템은 대화를 통해 사용자가 AI가 추론한 컬럼명을 수정할 수 있어야 한다.

#### Scenario: 사용자가 컬럼명 수정
- **WHEN** 사용자가 "col_2는 매출이 아니라 비용이야"라고 하면
- **THEN** 시스템은 컬럼명을 "cost" 또는 "비용"으로 업데이트해야 한다
- **AND** 변경 사항을 사용자에게 확인해야 한다

### Requirement: 데이터 타입 감지
시스템은 각 컬럼의 데이터 타입을 감지해야 한다: Date, Number, Category, Boolean.

#### Scenario: Date 타입 감지
- **WHEN** 컬럼이 날짜 패턴(YYYY-MM-DD, MM-DD-YYYY, Unix timestamp)과 일치하는 값을 포함하면
- **THEN** 시스템은 해당 컬럼을 Date 타입으로 표시해야 한다

#### Scenario: Number 타입 감지
- **WHEN** 컬럼이 숫자 값만 포함하면(선택적 통화 기호나 쉼표 포함)
- **THEN** 시스템은 해당 컬럼을 Number 타입으로 표시해야 한다

#### Scenario: Category 타입 감지
- **WHEN** 컬럼이 낮은 카디널리티를 가진 문자열 값을 포함하면(고유값이 전체 행의 10% 미만)
- **THEN** 시스템은 해당 컬럼을 Category 타입으로 표시해야 한다
